## 2.1 算力需求的指数级增长

### Transformer引发的算力革命

2017年，Google发布论文《Attention Is All You Need》，提出了Transformer架构。这个看似简单的"自注意力机制"，彻底改变了NLP领域，也引发了一场算力需求的核爆。

**模型参数增长趋势**:

| 年份 | 模型 | 参数量 | 训练算力 |
|------|------|--------|----------|
| 2018 | BERT-Base | 110M | 64 TPUs × 4天 |
| 2019 | GPT-2 | 1.5B | - |
| 2020 | GPT-3 | 175B | 数千V100 × 数月 |
| 2022 | PaLM | 540B | 6144 TPUv4 × 数月 |
| 2023 | GPT-4 | 估计1.8T | 估计上万A100 |

**每18个月，模型参数增长10倍。** 这不是摩尔定律，这是AI时代的"新摩尔定律"。

### 算力经济学：每美元能买多少FLOPS？

AI公司面临一个残酷的现实：**算力成本是最大开支**。

OpenAI训练GPT-3的成本估计在**1200万美元**以上。GPT-4的成本可能高达**1亿美元**。

## 2.2 GPU霸主：NVIDIA的生态系统

### CUDA：软件护城河

NVIDIA的GPU之所以在AI领域一家独大，不是因为硬件性能（虽然也很强），而是因为**CUDA生态系统**。

CUDA（Compute Unified Device Architecture）是NVIDIA推出的并行计算平台。

**为什么CUDA如此重要？**

1. **编程友好**：用C/C++就能写GPU程序
2. **工具链完善**：profiler、debugger、profiler一应俱全
3. **库丰富**：cuDNN、cuBLAS、cuFFT……开箱即用
4. **社区活跃**：Stack Overflow、GitHub上有大量资源

**迁移成本极高**：一个用CUDA优化的深度学习框架，要迁移到AMD ROCm或Intel oneAPI，工作量巨大。

### GPU架构演进

**Volta架构（V100，2017）**
- 第一代Tensor Core
- 支持FP16混合精度训练
- 16GB HBM2显存
- 125 TFLOPS (FP16)

**Ampere架构（A100，2020）**
- 第三代Tensor Core
- 支持稀疏化加速
- 40/80GB HBM2显存
- 312 TFLOPS (FP16)

**Hopper架构（H100，2022）**
- 第四代Tensor Core
- Transformer Engine（专门为Transformer优化）
- 80GB HBM3显存
- 989 TFLOPS (FP16)

**Blackwell架构（B200，2024）**
- 第二代Transformer Engine
- NVLink 5.0（1.8TB/s）
- 192GB HBM3e显存
- 20 petaFLOPS (FP4)

（此处需要一张图：NVIDIA GPU架构演进时间线）

### DGX与HGX：企业级AI基础设施

**NVIDIA DGX**：整机交付的AI超级计算机
- DGX A100：8×A100，640GB显存
- DGX H100：8×H100，640GB显存
- DGX B200：8×B200，1.5TB显存

**NVIDIA HGX**：模块化设计，OEM厂商集成
- HGX A100/H100/B200
- 灵活配置，适合云厂商大规模部署

**价格**：一台DGX H100售价约**40万美元**。这不是普通企业能随便买的设备。

## 2.3 挑战者：Google TPU与AWS Trainium

### Google TPU：专为AI而生

TPU（Tensor Processing Unit）是Google自研的AI专用芯片。

**设计哲学**：**
- 不像GPU那样通用，而是专门为矩阵运算优化
- 大规模片上内存（HBM），减少数据搬运
- 脉动阵列（Systolic Array）架构，高效利用带宽

**TPU版本演进**：

| 版本 | 时间 | 性能 | 特点 |
|------|------|------|------|
| TPU v1 | 2016 | 92 TFLOPS | 仅推理 |
| TPU v2 | 2017 | 180 TFLOPS | 训练+推理 |
| TPU v3 | 2018 | 420 TFLOPS | 液冷散热 |
| TPU v4 | 2021 | 275 TFLOPS | 3D torus网络 |
| TPU v5e | 2023 | 197 TFLOPS | 边缘优化 |
| TPU v5p | 2023 | 459 TFLOPS | 训练专用 |

**Google的策略**：**
- 不单独卖芯片，只通过Google Cloud提供服务
- 绑定TensorFlow和JAX生态
- 训练自己的大模型（PaLM、Gemini）

（此处需要一张图：TPU脉动阵列架构示意图）

### AWS Trainium与Inferentia

AWS推出了自研AI芯片：
- **Trainium**：训练专用
- **Inferentia**：推理专用

**Trainium2（2023）**：
- 比Trainium1快4倍
- 支持大规模分布式训练
- 通过AWS EC2 Trn2实例提供服务

**AWS的策略**：**
- 客户可以选择NVIDIA GPU或自研芯片
- 自研芯片价格更低（约30-40%成本节省）
- 但生态不如CUDA成熟

## 2.4 国产力量：华为昇腾与寒武纪

### 华为昇腾：全栈AI解决方案

**昇腾910B**：
- 7nm工艺
- 达芬奇架构（Da Vinci Architecture）
- 256 TFLOPS (FP16)
- 32GB HBM2显存

**与A100对比**：**
| 指标 | 昇腾910B | A100 |
|------|----------|------|
| FP16算力 | 256 TFLOPS | 312 TFLOPS |
| 显存 | 32GB | 40/80GB |
| 功耗 | 310W | 400W |
| 互联带宽 | - | 600GB/s |

**昇腾生态**：**
- **CANN**：对标CUDA的计算架构
- **MindSpore**：深度学习框架（对标PyTorch/TensorFlow）
- **ModelArts**：AI开发平台

**优势**：
- 自主可控，不受制裁影响
- 能效比优秀
- 国内支持好

**劣势**：
- 生态不如CUDA成熟
- 迁移成本较高
- 大规模集群经验相对较少

（此处需要一张图：昇腾达芬奇架构）

### 寒武纪：云端与终端并重

**思元370（MLU370）**：
- 7nm工艺
- 训练+推理两用
- 48 TOPS (INT8)

**寒武纪策略**：**
- 云端：思元系列（对标GPU）
- 终端：Cambricon-1N/1M（手机、IoT）
- 车端：与多家车企合作

**优势**：**
- 终端市场领先
- 与手机芯片厂商合作紧密

**劣势**：**
- 云端市场占有率较低
- 软件生态建设需要时间

## 2.5 多元算力架构设计

### 异构计算：不是选择，而是必须

现代AI基础设施很少只用一种芯片，而是**CPU + GPU + 专用加速器**的组合。

**为什么需要异构？**

1. **任务特性不同**：
   - 数据预处理：CPU更合适
   - 模型训练：GPU/TPU
   - 模型推理：根据延迟要求选择

2. **成本优化**：
   - 不是所有任务都需要高端GPU
   - 合理搭配可以降低成本

3. **供应安全**：
   - 避免单一供应商依赖
   - 国产化替代需求

### 算力调度与资源池化

**算力池化的核心思想**：**
把不同类型的算力资源统一管理，按需分配给不同的任务。

**调度策略**：**
- **优先级调度**：紧急任务优先获得资源
- **公平调度**：保证每个用户都能获得算力
- **抢占式调度**：高优先级任务可以抢占低优先级任务的资源
- **弹性调度**：根据负载自动扩缩容

**资源隔离**：**
- MIG（Multi-Instance GPU）：把一块GPU虚拟成多个实例
- 时间片调度：多个任务分时使用GPU
- 容器隔离：不同任务在独立容器中运行

（此处需要一张图：异构算力池化架构）

### 选型决策框架

企业在选择算力方案时，需要考虑以下因素：

**1. 任务类型**
- 训练 vs 推理
- 延迟敏感 vs 吞吐量优先
- 模型规模（参数、显存需求）

**2. 成本约束**
- CAPEX（采购成本）
- OPEX（运营成本：电费、维护）
- 云 vs 自建

**3. 生态兼容**
- 现有代码框架（PyTorch/TensorFlow/MindSpore）
- 团队技术栈
- 第三方库依赖

**4. 供应链安全**
- 供应商稳定性
- 地缘政治风险
- 国产化要求

**决策矩阵**：

| 场景 | 推荐方案 | 理由 |
|------|----------|------|
| 大模型训练 | NVIDIA A100/H100 | 生态成熟，性能最强 |
| 推理服务（云端） | T4/A10/A100 | 成本低，部署灵活 |
| 推理服务（边缘） | Jetson/昇腾310 | 功耗低，体积小 |
| 国产化替代 | 昇腾910B | 自主可控，性能可接受 |
| 成本敏感 | AWS Trainium/昇腾 | 价格优势明显 |

## 案例：字节跳动的多元算力实践

字节跳动的推荐系统每天处理数百亿次请求，需要海量算力支持。

**他们的策略**：**

1. **训练阶段**：
   - 主要使用NVIDIA A100/H100
   - 大规模分布式训练（数千张GPU）

2. **推理阶段**：**
   - 云端：A10/T4 GPU
   - 边缘：自研芯片 + 寒武纪
   - 端侧：手机NPU

3. **国产化探索**：**
   - 与华为合作，测试昇腾910B
   - 部分业务迁移到MindSpore

**效果**：**
- 整体成本降低20-30%
- 供应链风险分散
- 推理延迟降低40%

## 本章小结

这一章我们探讨了AI时代的算力博弈：

- **算力需求爆炸**：Transformer引发参数增长，每18个月10倍
- **NVIDIA霸主地位**：CUDA生态形成护城河，DGX/HGX企业级方案
- **挑战者崛起**：Google TPU专用架构，AWS Trainium成本优势
- **国产力量**：华为昇腾全栈方案，寒武纪云端+终端并重
- **多元算力架构**：异构计算、资源池化、智能调度成为标配

**关键洞察**：**
没有最好的芯片，只有最适合的芯片。企业需要根据自身业务特点、成本约束、生态兼容等因素，选择最优的算力组合。

在下一章，我们将探讨算力激增带来的另一个挑战——**散热**。当单块GPU功耗达到700W，传统的风冷已经无能为力，液冷技术成为必然选择。
